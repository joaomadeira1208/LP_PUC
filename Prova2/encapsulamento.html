Encapsulamento

  • Sistemas de programação de grande porte são constituídos de módulos
  • Módulo -> qualquer unidade de programa que possua um nome e que possa ser implementada como uma entidade independente
  • Módulo bem projetado tem um único objetivo e possui uma interface pequena com outros módulos -> REÚTILIZAVEL
  • Módulo encapsula seus componentes (tipos, constantes, variáveis, procedimentos, funções)
    • Componentes exportáveis -> componentes que são visíveis externamente ao módulo
    • Componentes ocultos -> componentes que são usados somente para auxiliar a implementação dos componentes exportáveis
  • Chave para modularidade -> abstração

    • Pacotes

  • Pacotes Simples
    • Um pacote é especificado através de uma lista de informações declarativas (qualquer denotável: tipos, constantes, variáveis, procedimentos, funções e outros pacotes)
    • Pode ser visto como um conjunto encapsulado de amarrações
  • Ocultamento de informação
    • Um pacote contém declarações de componentes exportáveis e escondidos. A distinção em Ada, é feita através da divisão de um pacote em duas partes:
      • Declaração do pacote - declara somente os componentes exportáveis
      • Corpo do pacote - contém declarações de todos os componentes escondidos, além dos corpos de procedimentos e funções exportáveis
    • Declaração do pacote contém somente as informações necessária ao usuário para que ele possa utilizar o pacote
